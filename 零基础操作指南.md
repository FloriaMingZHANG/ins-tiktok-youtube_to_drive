# 零基础操作指南：从 Google 表格下载 Ins 视频并上传到 Drive

> **费用说明**：本方案用到的全部工具均为**免费**  
> - Python：免费开源  
> - Google 表格 / Google Drive / Google Cloud（本用途下的用量）：免费额度内使用，个人批量下载上传不会收费  
> - yt-dlp、gspread 等程序库：免费开源  
> 无需购买任何软件或订阅。

---

## 一、你需要准备的东西

- 一台电脑（Windows 或 Mac 均可）
- 一个 Google 账号（用来建表格、Drive、Cloud 项目）
- 已经有一个 **Google 表格**，里面有一列是 Instagram 链接，一列是你要起的文件名（下面会具体说格式）
- 能打开「终端」（Mac）或「命令提示符 / PowerShell」（Windows）

## 二、第一步：安装 Python

Python 是一种免费编程语言，我们的脚本需要它才能运行。

### 2.1 下载 Python

1. 打开浏览器，访问：**https://www.python.org/downloads/**
2. 点击黄色的 **「Download Python 3.x.x」** 按钮下载安装包。
3. 运行下载好的安装程序。

### 2.2 安装时请注意（重要）

- **Windows**：在安装界面**务必勾选**「**Add Python to PATH**」，然后点「Install Now」。
- **Mac**：按提示一路「继续」即可。

安装完成后**关闭并重新打开**终端/命令提示符，再进行下一步。

### 2.3 检查是否安装成功

- **Windows**：按 `Win + R`，输入 `cmd` 回车，在黑色窗口里输入：  
  `python --version`  
  回车后若显示类似 `Python 3.12.x` 就说明安装成功。
- **Mac**：打开「终端」（在「启动台」或「应用程序」里找），输入：  
  `python3 --version`  
  若显示类似 `Python 3.12.x` 就说明安装成功。

如果提示“找不到 python”或“不是内部或外部命令”，说明安装时没勾选「Add Python to PATH」，请卸载后重新安装并勾选该项。

## 三、第二步：找到项目文件夹并打开「终端」

我们的脚本在一个叫 **ins_to_drive** 的文件夹里。

### 3.1 找到 ins_to_drive 文件夹

- 你之前把 Cursor 的工作区放在：  
  `OneDrive-bfsu.edu.cn/cursor`  
- 在这个 **cursor** 文件夹里，会有一个子文件夹叫 **ins_to_drive**。  
- 记下它的完整路径，例如：  
  - Mac：`/Users/你的用户名/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive`  
  - Windows：`C:\Users\你的用户名\OneDrive - bfsu.edu.cn\cursor\ins_to_drive`  
  （你的实际路径以你自己电脑为准。）

### 3.2 在「终端」里进入这个文件夹

- **Mac**：打开「终端」，输入（把路径改成你的实际路径）：  
  `cd /Users/你的用户名/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive`  
  回车。
- **Windows**：打开「命令提示符」或「PowerShell」，输入（把路径改成你的实际路径）：  
  `cd "C:\Users\你的用户名\OneDrive - bfsu.edu.cn\cursor\ins_to_drive"`  
  回车。

看到路径前面出现 `ins_to_drive` 或类似字样，说明已经进入正确文件夹。**后面的所有命令都在这个文件夹下执行。**

## 四、第三步：安装脚本需要的「依赖」（免费程序库）

在**同一个终端窗口**、**已经 cd 到 ins_to_drive 文件夹**的情况下，依次输入下面两条命令（每输入一条就按回车，等执行完再输入下一条）。

**第一条：**

- Mac：  
  `python3 -m venv .venv`  
- Windows：  
  `python -m venv .venv`

**第二条（创建虚拟环境后）：**

- Mac：  
  `source .venv/bin/activate`  
- Windows：  
  `.venv\Scripts\activate`

执行成功后，命令行前面可能会出现 `(.venv)`，表示已经在这个项目的环境里。

**第三条：安装依赖**

- Mac：  
  `pip install -r requirements.txt`  
- Windows：  
  `pip install -r requirements.txt`

等待安装结束，没有报错即可。**这些库都是免费的。**

## 五、第四步：在 Google 云端创建「服务账号」并拿到密钥（免费）

这一步是让 Google 允许我们的脚本读取你的表格、往你的 Drive 上传文件。全部在 Google 免费额度内完成。

### 5.1 打开 Google Cloud 控制台

1. 用浏览器打开：**https://console.cloud.google.com/**
2. 用你的 Google 账号登录。

### 5.2 创建项目（如果还没有）

1. 点击顶部的「选择项目」→「新建项目」。
2. 项目名称随便填，比如「Ins 下载」。
3. 点击「创建」。创建好后，确保顶部选中的是这个新项目。

### 5.3 开启两个 API（表格 + 网盘）

1. 左侧菜单点「**API 和服务**」→「**库**」（或「已启用的 API 和服务」旁边的「启用 API 和服务」）。
2. 在搜索框输入 **Google Sheets API**，点进去，点击「**启用**」。
3. 再回到「库」，搜索 **Google Drive API**，点进去，点击「**启用**」。

### 5.4 创建服务账号并下载密钥

1. 左侧菜单点「**API 和服务**」→「**凭据**」。
2. 点击「**+ 创建凭据**」→「**服务账号**」。
3. 服务账号名称可随便填（如「ins-downloader」），点「创建并继续」→ 角色可先跳过或选「编辑者」→「完成」。
4. 在「凭据」页面的「服务账号」列表里，点击刚创建的那个账号（邮箱形如 `xxx@项目名.iam.gserviceaccount.com`）。
5. 打开「**密钥**」标签 →「**添加密钥**」→「**创建新密钥**」→ 类型选「**JSON**」→「创建」。  
   浏览器会下载一个 JSON 文件。

### 5.5 把密钥放到脚本所在文件夹

1. 把刚下载的 JSON 文件**复制**到 **ins_to_drive** 文件夹里（就是放 main.py 的那个文件夹）。
2. 把该文件**重命名**为：**credentials.json**（注意是英文、没有空格）。  
   最终 ins_to_drive 文件夹里要有一个文件叫 `credentials.json`。

**重要**：这个文件里包含一个邮箱（`client_email`），长得像：  
`ins-downloader@你的项目名.iam.gserviceaccount.com`  
**请把这个邮箱复制下来**，下一步会用到。

## 六、第五步：把表格和 Drive 文件夹「共享」给服务账号

脚本只能访问你「共享」给它的表格和文件夹，这样才安全。

### 6.1 共享 Google 表格

1. 打开你的 **Google 表格**（就是放 Ins 链接和文件名的那张表）。
2. 点击右上角「**共享**」。
3. 在「添加用户和组」里**粘贴**上一步复制的服务账号邮箱（`xxx@xxx.iam.gserviceaccount.com`）。
4. 权限选「**查看者**」即可（脚本只读表格）。
5. 取消勾选「通知用户」（服务账号不会收邮件），点「共享」。

### 6.2 共享要上传到的 Drive 文件夹（可选）

- 如果希望视频上传到 **某个具体文件夹**：  
  1. 在 Google Drive 网页打开那个文件夹。  
  2. 右键该文件夹 →「共享」→ 同样添加服务账号邮箱，权限选「**编辑者**」。  
- 如果希望上传到「我的云端硬盘」根目录：可以不共享文件夹，脚本会上传到根目录（需在 .env 里不填 DRIVE_FOLDER_ID，见下文）。

## 七、第六步：准备好你的 Google 表格格式

表格建议这样排（示例）：

| A 列（Ins 链接） | B 列（文件名） |
|------------------|----------------|
| https://www.instagram.com/reel/xxxxx/ | 我的第一个视频 |
| https://www.instagram.com/p/yyyyy/    | 我的第二个视频 |

- **第一行**：可以是表头（如「链接」「文件名」），脚本会从第二行开始读数据。
- **一列**：必须是完整的 Instagram 链接（帖子或 Reels 都可以）。
- **另一列**：你想起的文件名（不要写 `.mp4` 也可以，程序会自动加）。

记下：  
- 链接在第几列（A/B/C…）→ 后面填到 `URL_COLUMN`  
- 文件名在第几列 → 后面填到 `NAME_COLUMN`  
- 表头有几行（通常 1 行）→ 后面填到 `HEADER_ROWS`

## 八、第七步：创建并填写配置文件 .env

脚本通过一个叫 **.env** 的配置文件知道：用哪张表、哪几列、上传到哪个文件夹。**不需要会编程**，只要会复制、改字即可。

### 8.1 复制示例配置

1. 在 **ins_to_drive** 文件夹里，找到 **config.example.env** 这个文件。
2. **复制**一份，并**重命名**为：**.env**  
   （前面有一个点，后面是 env；若系统提示「无法重命名」可先改成 `env.txt`，再改成 `.env`，或使用记事本「另存为」保存为 `.env`。）

### 8.2 用记事本打开 .env 并修改

用「记事本」（Windows）或「文本编辑」（Mac）打开 **.env**，按下面说明改（每行前面的 # 表示注释，可以不改）：

```env
# 你的 Google 表格的完整网址，从浏览器地址栏复制
SPREADSHEET_URL=https://docs.google.com/spreadsheets/d/这里是一串字母数字/edit

# 如果表格有多个「工作表」（左下角有多个标签），填工作表名字；只有一张表就留空
SHEET_NAME=

# 链接在第几列？A 列就写 A，B 列就写 B
URL_COLUMN=A
# 文件名在第几列？
NAME_COLUMN=B

# 表头有几行？通常第一行是表头就写 1
HEADER_ROWS=1

# 要上传到某个 Drive 文件夹时：在浏览器打开该文件夹，看地址栏，
# 找到 /folders/ 后面那一串，复制到这里；上传到根目录就留空
DRIVE_FOLDER_ID=
```

保存并关闭 .env。

## 九、第八步：运行脚本

1. 确认：  
   - 已经用 **cd** 进入 **ins_to_drive** 文件夹；  
   - 已经执行过 **source .venv/bin/activate**（Mac）或 **.venv\Scripts\activate**（Windows），前面有 `(.venv)`；  
   - **credentials.json** 在 ins_to_drive 里；  
   - **.env** 已按你的表格和 Drive 填好并保存。
2. 在终端输入：  
   **Mac**：`python3 main.py`  
   **Windows**：`python main.py`  
   回车。
3. 脚本会：  
   - 按行读表格；  
   - 依次下载每条 Ins 链接的视频；  
   - 用你写的那一列命名并上传到 Google Drive（或你指定的文件夹）。

过程中会在终端里看到进度；若有某一条下载失败，会提示，但会继续处理后面的行。

## 十、常见问题（不用懂编程也能对照检查）

| 现象 | 可能原因 | 你可以做的 |
|------|----------|------------|
| 提示「请设置 SPREADSHEET_URL 或 SPREADSHEET_KEY」 | .env 里没填表格地址 | 在 .env 里填上 SPREADSHEET_URL=你的表格完整网址 |
| 提示打不开表格 / 没有权限 | 表格没共享给服务账号 | 按「第六步」把表格共享给 credentials.json 里的那个邮箱 |
| 上传失败 / 找不到文件夹 | 文件夹没共享或 ID 填错 | 把 Drive 文件夹共享给同一邮箱，并检查 .env 里 DRIVE_FOLDER_ID 是否从地址栏 /folders/ 后面复制正确 |
| 上传报错「Service Accounts do not have storage quota」 | 服务账号没有网盘空间，不能上传到「我的云端硬盘」 | 用**你自己的账号**上传：按项目里的 **上传到我的网盘-OAuth配置.md** 配置 OAuth（下载 client_secret.json 并重命名放到 ins_to_drive），再运行脚本，在浏览器里登录一次即可 |
| Ins 链接下载失败 | 链接无效或需要登录 | 先在浏览器里打开链接看能否播放；若需登录，可查 README 里「COOKIES_FILE」的说明（进阶） |
| 终端里提示「python 不是内部或外部命令」 | Python 没装好或没选「Add to PATH」 | 重新安装 Python，勾选「Add Python to PATH」后重启终端 |
| 提示找不到某个模块（如 gspread） | 依赖没装或没激活环境 | 确认先执行了 activate，再执行 pip install -r requirements.txt |
