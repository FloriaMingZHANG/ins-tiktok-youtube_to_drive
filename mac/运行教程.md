# Mac 正常运行教程（`mac/app` 版本）

下面按**第一次使用** → **以后每次运行**的顺序写。路径示例用你的仓库位置，若不在同一目录请整体替换。

**项目根目录**（示例）：

```text
/Users/floriazhang/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive
```

**Mac 程序与配置所在目录**：

```text
.../ins_to_drive/mac/app
```

---

## 一、你需要先具备什么

1. **Python 3**  
   终端执行 `python3 --version`，能显示 3.x 即可。没有则安装： [python.org](https://www.python.org/downloads/) 或 `brew install python`。

2. **Google 云端**（一次性准备）  
   - 在 [Google Cloud Console](https://console.cloud.google.com/) 启用 **Google Sheets API**、**Google Drive API**。  
   - 创建**服务账号**，下载密钥 JSON，保存为 **`mac/app/credentials.json`**（或通过下文「复制脚本」从根目录拷过来）。  
   - 把**表格**、**目标 Drive 文件夹**共享给该 JSON 里的 **`client_email`**（表格一般「查看者」，文件夹「编辑者」）。

3. **ffmpeg（按需）**  
   若要从视频里截封面：`brew install ffmpeg`。

4. **Cookie（按需）**  
   下载 Instagram、X 等需登录的平台时，在 **`mac/app/.env`** 里配置 `COOKIES_FILE` 或 `COOKIES_FROM_BROWSER`（详见仓库根目录 **README.md**）。

## 二、第一次使用（按顺序做一遍）

### 步骤 1：打开项目

用 **Cursor**（或 VS Code）：**文件 → 打开文件夹**，打开 **`ins_to_drive/mac/app`** 或整个 **`ins_to_drive`** 均可。

### 步骤 2：把密钥和配置放进 `mac/app/`

**方式 A（推荐）**：根目录里若已有 `credentials.json`、`.env` 等，在项目里执行：

```bash
bash "/Users/floriazhang/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive/mac/sync_secrets_to_app.sh"
```

**方式 B**：手动把 `credentials.json` 放进 `mac/app/`，并在 `mac/app` 下执行：

```bash
cd "/Users/floriazhang/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive/mac/app"
cp config.example.env .env
```

再用编辑器打开 **`mac/app/.env`**，至少确认这些变量（字母按你表格实际列）：

| 变量 | 含义 |
|------|------|
| `SPREADSHEET_URL` | Google 表格完整链接 |
| `URL_COLUMN` | 视频链接列，如 `A` |
| `NAME_COLUMN` | 文件名列，如 `B` |
| `RESULT_COLUMN` | 写回 Drive 视频链接的列，如 `C` |
| `DRIVE_FOLDER_ID` | 目标文件夹 ID（Drive 链接里 `/folders/` 后面一段） |

其余项（封面列、飞书、慢网等）按根目录 **README.md** 说明填写。

### 步骤 3：安装 Python 依赖（虚拟环境）

在终端执行（路径按你的实际仓库改）：

```bash
bash "/Users/floriazhang/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive/mac/setup_mac.sh"
```

完成后，`mac/app/` 下会出现 **`.venv`**，依赖装在虚拟环境里，**不污染**系统 Python。

## 三、以后「正常运行」的两种方式（二选一）

### 方式 1：进目录 + 激活环境（适合经常改 `.env`、跑多个脚本）

```bash
cd "/Users/floriazhang/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive/mac/app"
source .venv/bin/activate
```

然后执行主程序（见下一节）。  
用完后可执行 `deactivate` 退出虚拟环境（可选）。

### 方式 2：一条命令跑主程序（不用先 `cd`）

```bash
bash "/Users/floriazhang/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive/mac/run_main.sh"
```

需要在后面加参数时，例如只跑视频：

```bash
bash "/Users/floriazhang/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive/mac/run_main.sh" video
```

## 四、主程序 `main.py` 怎么跑

在 **已激活虚拟环境** 且 **当前目录为 `mac/app`** 时：

| 命令 | 作用 |
|------|------|
| `python main.py` | 按 **`.env` 里 `DO_VIDEO` / `DO_COVER`** 的设置执行 |
| `python main.py video` | **只**下载并上传视频、写回视频链接 |
| `python main.py cover` | **只**处理封面（不按整段视频下载） |
| `python main.py both` | 视频 + 封面都做 |

使用 **方式 2** 时，请使用**真实路径**（下面二选一，**不要把 `...` 原样打进终端**）：

**从任意目录**（推荐复制整行，路径按你的电脑修改）：

```bash
bash "/Users/floriazhang/Library/CloudStorage/OneDrive-bfsu.edu.cn/cursor/ins_to_drive/mac/run_main.sh" cover
```

**若当前目录已经是 `mac/app`**（提示符里常见 `app %`）：

```bash
bash ../run_main.sh cover
```

同理可把末尾的 `cover` 换成 `video`、`both`，或不写参数。

运行过程中按终端提示操作即可（例如 Cookie、浏览器未关闭等提示）。

## 六、小结：最省事的「日常流程」

1. 打开终端。  
2. 执行 `bash "/你的路径/ins_to_drive/mac/run_main.sh"`（把 `/你的路径` 换成实际路径），或已进入 `mac/app` 且已 `source .venv/bin/activate` 时直接 `python main.py`。  
3. 改表格里的链接/文件名后，重复运行即可。

更细的故障排查见仓库根目录 **README.md** 里的「常见问题」。
